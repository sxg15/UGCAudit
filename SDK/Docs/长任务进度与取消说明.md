# 长任务进度与取消说明

本文档说明耗时模块怎样向客户端报告进度，以及用户取消任务时模块应该怎样停止。

## 什么时候需要进度

如果模块可能运行超过几秒，建议支持进度。

常见长任务包括：

- 批量图片处理。
- OCR。
- 模型推理。
- 文件夹扫描。
- 导出预览图或标注图。
- 调用外部 CLI。

没有进度时，模块可能仍在运行，但用户会以为它没有反应。

## progressPath

实时运行时，客户端会在 `input.json` 中传入：

```json
{
  "progressPath": "D:/UGCAuditData/runs/run_001/steps/step_1/progress.jsonl"
}
```

模块可以向这个文件追加 JSON 行：

```json
{"progress": 0.5, "message": "已处理 50 张图片", "processed": 50, "total": 100}
```

字段建议：

| 字段 | 说明 |
| --- | --- |
| `progress` | 0 到 1 之间的小数。 |
| `message` | 给用户看的当前状态。 |
| `processed` | 已处理数量。 |
| `total` | 总数量。 |

## 进度写法

Python 示例：

```python
import json
from pathlib import Path


def write_progress(input_data, progress, message, processed=None, total=None):
    path = str(input_data.get("progressPath", "")).strip()
    if not path:
        return
    item = {"progress": progress, "message": message}
    if processed is not None:
        item["processed"] = processed
    if total is not None:
        item["total"] = total
    with Path(path).open("a", encoding="utf-8") as handle:
        handle.write(json.dumps(item, ensure_ascii=False) + "\n")
```

建议：

- 开始时写一次 `progress: 0`。
- 循环处理中定期写进度。
- 完成前写一次 `progress: 1`。
- 不要每处理一个极小项目就写一次，避免日志过多。

## cancelPath

用户取消任务时，客户端会创建取消标记文件。路径在 `input.json` 中：

```json
{
  "cancelPath": "D:/UGCAuditData/runs/run_001/steps/step_1/cancel.flag"
}
```

模块应在循环中检查这个文件是否存在。

Python 示例：

```python
from pathlib import Path


def is_cancelled(input_data):
    path = str(input_data.get("cancelPath", "")).strip()
    return bool(path) and Path(path).exists()
```

## 取消时输出

收到取消后，模块应尽快写出 `output.json`：

```json
{
  "status": "cancelled",
  "verdict": "review",
  "message": "任务已中断。",
  "processedFiles": 50,
  "matchedFiles": 0,
  "artifactCount": 0,
  "reportSection": "### 模块已中断\n\n用户中断了任务，已保留当前已生成的产物。"
}
```

建议：

- 不要删除已经生成的有效产物。
- 报告里写清楚处理到哪里。
- 如果生成了部分图片或表格，仍可在 `outputs` 中写出已完成部分。
- 不要把取消当成通过。

## 环境变量兜底

部分模块也可以从环境变量读取路径：

| 环境变量 | 对应字段 |
| --- | --- |
| `UGCAUDIT_PROGRESS_FILE` | `progressPath` |
| `UGCAUDIT_CANCEL_FILE` | `cancelPath` |

如果同时存在，优先使用 `input.json` 中的字段。

## 常见问题

| 现象 | 原因 | 处理 |
| --- | --- | --- |
| 模块长时间没有反馈 | 没有写 `progressPath`。 | 在长循环中追加进度 JSON 行。 |
| 点击取消后还在运行 | 没有检查 `cancelPath`。 | 每个批处理循环检查一次取消标记。 |
| 取消后结果显示失败 | 没有写出 `output.json`。 | 取消时也写合法输出。 |
| 进度一直不变 | `progress` 没有递增，或写入不是 JSON 行。 | 确认每行都是独立 JSON。 |

## 验收标准

长任务交付前应验证：

- 运行开始后能看到进度。
- 处理中进度会变化。
- 点击取消后模块能尽快停止。
- 取消后有合法 `output.json`。
- 已生成的产物仍能在报告或输出目录中找到。
