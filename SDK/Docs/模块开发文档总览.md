## 开发者必须知道的约定

- 一个模块就是一个文件夹。
- 模块文件夹根目录必须有 `module.json`。
- 客户端导入模块时读取 `module.json`，不会把大型模型目录复制到客户端内部目录。
- 导入后的模块源文件夹要继续保留在原位置。
- 模块运行时，客户端会生成 `input.json`，模块读取它。
- 模块结束前，必须写出 `output.json`。
- 最终报告由客户端汇总生成，模块只写自己的 `reportSection` 片段。
- 模块不要写死某台机器上的绝对路径，运行所需路径都从 `input.json` 或参数里读取。
- 耗时模块要写进度，并在用户取消时尽快写出合法结果后退出。
- 交付前要用示例输入、本地试跑、客户端运行或 CLI 运行做至少一次验收。

## 推荐开发流程

1. 先用 [模块开发快速开始](./模块开发快速开始.md) 做出最小模块。
2. 按 [模块文件夹与配置说明](./模块文件夹与配置说明.md) 补齐图标、参数和模型目录。
3. 按 [模块输入与输出说明](./模块输入与输出说明.md) 确认输入输出字段。
4. 按 [模块类型与流程连接说明](./模块类型与流程连接说明.md) 把模块接到流程里。
5. 按 [模块依赖与运行环境说明](./模块依赖与运行环境说明.md) 清理路径、依赖和模型配置。
6. 如果模块耗时，按 [长任务进度与取消说明](./长任务进度与取消说明.md) 加进度和取消处理。
7. 按 [模块调试与问题排查](./模块调试与问题排查.md) 在客户端跑一遍。
8. 按 [模块自动化验收与 CLI 说明](./模块自动化验收与CLI说明.md) 做交付前验收。
9. 按 [模块打包与交付说明](./模块打包与交付说明.md) 清理临时文件并交付。
