# 模块依赖与运行环境说明

本文档说明模块运行时的 Python、依赖、模型目录和路径约定。

## Python 运行方式

如果 `launch.command` 指向 `.py` 文件，客户端会用自己的 Python 运行它。

推荐配置：

```json
{
  "launch": {
    "launchType": "python",
    "command": "main.py",
    "args": ["--input", "{inputJson}", "--output", "{outputJson}"],
    "notes": "使用客户端运行环境执行 main.py。"
  }
}
```

注意：

- `launchType` 主要用于界面显示。
- 真正决定 Python 启动的是入口文件扩展名 `.py`。
- 入口文件相对路径会按模块文件夹解析。
- 模块运行时当前目录是模块文件夹。
- 非 `.py` 入口必须是可以直接执行的文件。

## 命令参数

建议始终显式写：

```json
{
  "args": ["--input", "{inputJson}", "--output", "{outputJson}"]
}
```

如果没有写 `{inputJson}` 或 `{outputJson}`，客户端会自动补上。但显式写出更方便本地调试，也能避免自定义入口参数顺序不清楚。

可用占位符：

| 占位符 | 含义 |
| --- | --- |
| `{inputJson}` | 客户端生成的输入文件路径。 |
| `{outputJson}` | 模块必须写出的输出文件路径。 |
| `{resourceRoot}` | 本次运行的资源根目录。 |
| `{paramsJson}` | 当前节点参数 JSON 字符串。 |

## 运行时环境变量

模块可以优先读取 `input.json`。如果需要，也可以读取这些环境变量：

| 环境变量 | 含义 |
| --- | --- |
| `UGCAUDIT_RUN_ID` | 本次运行 ID。 |
| `UGCAUDIT_TASK_NAME` | 本次任务名称。 |
| `UGCAUDIT_RESOURCE_ROOT` | 本次运行资源根目录。 |
| `UGCAUDIT_ARTIFACT_DIR` | 本次任务产物目录。 |
| `UGCAUDIT_STEP_ARTIFACT_DIR` | 当前步骤产物目录。 |
| `UGCAUDIT_PROGRESS_FILE` | 进度文件路径。 |
| `UGCAUDIT_CANCEL_FILE` | 取消标记文件路径。 |
| `UGCAUDIT_MODULE_DIR` | 当前模块目录。 |

建议模块逻辑仍以 `input.json` 为准，环境变量只作为辅助。

## 依赖文件

轻量依赖可以在模块 README 中写清楚安装方式。重依赖不要在模块运行时自动下载。

推荐做法：

- 在 `requirements.txt` 写清 Python 包和版本。
- 在 README 写清是否需要 GPU、首次准备时间、模型来源。
- 如果依赖已经随模块交付，说明依赖目录和使用方式。
- 不要把 `.venv/`、下载缓存、临时日志放入交付包。

如果依赖缺失，模块应写出清晰错误：

```json
{
  "status": "error",
  "verdict": "error",
  "message": "缺少依赖：pillow。请先安装 requirements.txt。",
  "reportSection": "### 模块运行失败\n\n缺少依赖：pillow。"
}
```

## 模型目录

如果模块需要模型，优先使用：

```text
MyAuditModule/
  module.json
  main.py
  Model/
    ...
```

并在 `module.json` 中声明 `modelPath` 参数：

```json
{
  "key": "modelPath",
  "name": "模型目录",
  "description": "选择本地模型所在目录。",
  "parameterType": "path",
  "defaultValue": "",
  "required": true,
  "options": []
}
```

客户端发现模块目录下有 `Model/` 时，会把它作为默认模型目录。模块代码应从 `input.json` 的 `modelPath` 读取最终路径，不要写死 `Model` 的绝对路径。

## 路径规则

模块代码不要写死开发机路径，例如：

```text
D:/MyComputer/Models/xxx
C:/Users/me/Desktop/test
```

应从这些位置读取路径：

- `input.json` 的 `inputs`。
- `input.json` 的 `params`。
- `input.json` 的 `modelPath`。
- `input.json` 的 `artifactDir` 和 `stepArtifactDir`。
- 模块目录内的相对资源路径。

报告里的本地图片和本地文件链接建议使用绝对路径。相对路径只用于模块内部读取资源，不建议写到最终报告里。

## 交付前确认

交付前确认：

- 没有写死开发机绝对路径。
- 没有把 `.venv/`、缓存、日志、隐私数据放进模块目录。
- 模型路径从 `input.json.modelPath` 读取。
- 依赖缺失时能输出清楚错误，而不是只崩溃。
- 在另一台机器上移动模块目录后，仍能通过重新导入运行。
