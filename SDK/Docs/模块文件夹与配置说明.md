# 模块文件夹与配置说明

本文档说明模块目录应该怎么组织，以及 `module.json` 里每个字段怎么写。

## 标准目录结构

推荐结构：

```text
MyAuditModule/
  module.json
  main.py
  README.md
  icon.png
  Model/
    model-file.bin
  assets/
    labels.json
```

必需文件：

- `module.json`
- `launch.command` 指向的入口文件，例如 `main.py`

可选文件：

- `README.md`：给模块使用者看的说明。
- `icon.png`：模块图标。
- `Model/`：模型文件目录。
- `assets/`：标签、规则、模板等轻量资源。

## module.json 完整示例

```json
{
  "id": "custom.company.image-check",
  "name": "图片合规检测",
  "kind": "image_safety",
  "summary": "检测图片中的合规风险。",
  "modelLabel": "图片审核模型",
  "icon": "icon.png",
  "modelConfigured": false,
  "launch": {
    "launchType": "python",
    "command": "main.py",
    "args": ["--input", "{inputJson}", "--output", "{outputJson}"],
    "notes": "运行本地图片审核模型。"
  },
  "dataOutputs": [],
  "parameters": [
    {
      "key": "modelPath",
      "name": "模型目录",
      "description": "选择本地模型所在目录。",
      "parameterType": "path",
      "defaultValue": "",
      "required": true,
      "options": []
    },
    {
      "key": "threshold",
      "name": "风险阈值",
      "description": "大于等于该值时进入人工复审。",
      "parameterType": "number",
      "defaultValue": 0.75,
      "required": true,
      "options": []
    }
  ]
}
```

## 基础字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `id` | 是 | 模块唯一 ID。建议使用 `custom.公司名.模块名`。 |
| `name` | 是 | 客户端里显示的模块名称。 |
| `kind` | 是 | 模块类型，决定流程图端口和数据连接方式。 |
| `summary` | 否 | 模块简介，会显示在模块列表里。 |
| `modelLabel` | 否 | 模型路径在界面上的名称。没有模型时可写“无需模型”。 |
| `icon` | 是 | 图标名称或图片文件路径。 |
| `modelConfigured` | 否 | 没有模型依赖的模块可以写 `true`。 |
| `launch` | 是 | 模块启动方式。 |
| `parameters` | 否 | 模块参数，会显示在流程节点右侧参数区。 |
| `dataOutputs` | 否 | 模块要向后续步骤提供的数据输出口。 |

## 模块 ID 规则

推荐：

```text
custom.company.module-name
custom.team.text-risk
custom.vendor.ocr
```

不要使用：

- `system.*`：系统节点使用。
- 已存在的内置模块 ID。
- 空格、斜杠、反斜杠等不适合作为目录名的字符。

客户端会按模块 ID 识别模块。重新导入相同 ID 的模块时，会用新的模块文件夹记录替换旧记录。

## 模块类型 kind

常用自定义模块类型：

| `kind` | 用途 | 数据输入 | 数据输出 |
| --- | --- | --- | --- |
| `image_ocr` | 图片文字识别 | 图片集合 | 文本集合 |
| `image_safety` | 图片合规检测 | 图片集合 | 无 |
| `text_safety` | 文本合规检测 | 文本集合 | 无 |
| `folder_processor` | 文件夹处理 | 文件夹 | 默认无；可通过 `dataOutputs` 声明图片、文本或文件夹输出 |

如果填写客户端不认识的 `kind`，模块通常只有顺序输入和顺序输出，不会自动出现图片、文本或文件夹数据口。

## 图标 icon

`icon` 支持两种写法。

第一种是内置图标名称：

```json
{
  "icon": "file-check"
}
```

当前常用内置图标名：

```text
play-circle
file-output
scan-text
shield-alert
database
folder-open
file-text
hard-drive
group
sticky-note
file-check
```

第二种是图片文件：

```json
{
  "icon": "icon.png"
}
```

图片路径可以是相对模块文件夹的路径。支持常见图片格式，例如 `png`、`jpg`、`webp`、`svg`、`ico`。如果写了图片路径，文件必须存在，否则模块会导入失败。

## 启动配置 launch

Python 模块推荐写法：

```json
{
  "launch": {
    "launchType": "python",
    "command": "main.py",
    "args": ["--input", "{inputJson}", "--output", "{outputJson}"],
    "notes": "使用客户端自带 Python 运行。"
  }
}
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `launchType` | 给人看的启动类型说明，推荐写 `python` 或 `exe`。 |
| `command` | 启动文件。相对路径会按模块文件夹解析。 |
| `args` | 启动参数。 |
| `notes` | 在模块管理页显示的启动说明。 |

占位符：

| 占位符 | 含义 |
| --- | --- |
| `{inputJson}` | 客户端生成的输入文件路径。 |
| `{outputJson}` | 模块必须写出的输出文件路径。 |
| `{resourceRoot}` | 本次运行的资源根目录。 |
| `{paramsJson}` | 当前节点参数的 JSON 字符串。 |

如果入口文件是 `.py`，客户端会用运行环境里的 Python 启动它。非 `.py` 文件必须是可以直接执行的文件。

PowerShell 示例：

```json
{
  "launch": {
    "launchType": "exe",
    "command": "C:/Windows/System32/WindowsPowerShell/v1.0/powershell.exe",
    "args": [
      "-NoProfile",
      "-ExecutionPolicy",
      "Bypass",
      "-File",
      "runner.ps1",
      "-InputPath",
      "{inputJson}",
      "-OutputPath",
      "{outputJson}"
    ],
    "notes": "使用 PowerShell 脚本运行。"
  }
}
```

## 参数 parameters

参数会显示在流程节点右侧。模块运行时，用户填写的参数会出现在 `input.json` 的 `params` 字段里。

参数通用格式：

```json
{
  "key": "threshold",
  "name": "风险阈值",
  "description": "大于等于该值时进入人工复审。",
  "parameterType": "number",
  "defaultValue": 0.75,
  "required": true,
  "options": []
}
```

支持的 `parameterType`：

| 类型 | 界面表现 | `defaultValue` 示例 |
| --- | --- | --- |
| `string` | 单行文本 | `"zh"` |
| `number` | 数字输入 | `0.75` |
| `boolean` | 勾选框 | `true` |
| `select` | 单选下拉框 | `"medium"` |
| `multiSelect` | 多选勾选框 | `["violence", "ad"]` |
| `stringList` | 字符串列表 | `["广告", "联系方式"]` |
| `policyList` | 策略列表 | `[{"name": "广告", "description": "推广内容"}]` |
| `path` | 本地路径输入 | `""` |
| `textarea` | 多行文本 | `"规则说明"` |

## 数据输出口 dataOutputs

如果模块运行后要把生成的数据继续交给下游模块，可以在 `module.json` 中声明 `dataOutputs`：

```json
{
  "dataOutputs": [
    {
      "handle": "images",
      "name": "图片",
      "dataType": "imageCollection"
    }
  ]
}
```

支持的 `dataType`：

| 类型 | 含义 |
| --- | --- |
| `imageCollection` | 图片集合 |
| `textCollection` | 文本集合 |
| `folder` | 文件夹 |

模块运行结束时，需要在 `output.json` 的 `outputs.<handle>` 下写出对应数据。图片集合示例见 [模块输入与输出说明](./模块输入与输出说明.md)。

`select` 和 `multiSelect` 需要提供 `options`：

```json
{
  "key": "level",
  "name": "审核等级",
  "description": "选择审核严格程度。",
  "parameterType": "select",
  "defaultValue": "normal",
  "required": true,
  "options": [
    { "label": "宽松", "value": "loose" },
    { "label": "正常", "value": "normal" },
    { "label": "严格", "value": "strict" }
  ]
}
```

## 模型目录 Model

如果模块文件夹下有 `Model/` 目录，客户端会把它作为模块的默认模型目录。

推荐结构：

```text
MyAuditModule/
  module.json
  main.py
  Model/
    config.json
    model.bin
```

如果模块需要模型路径，建议提供一个 key 为 `modelPath` 的参数：

```json
{
  "key": "modelPath",
  "name": "模型目录",
  "description": "选择本地模型所在目录。",
  "parameterType": "path",
  "defaultValue": "",
  "required": true,
  "options": []
}
```

这样模块运行时可以直接从 `input.json` 的 `modelPath` 字段读取路径。
