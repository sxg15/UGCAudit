# 模块类型与流程连接说明

本文档说明 `kind` 会怎样影响流程图端口，以及模块应该怎么连到流程里。

## 两类连线

流程图里有两类连线：

| 连线 | 作用 |
| --- | --- |
| 顺序线 | 决定步骤什么时候运行。 |
| 数据线 | 决定图片集合、文本集合传给谁。 |

一个模块通常同时需要顺序线和数据线。只有顺序线但没有数据线时，模块可能能启动，但拿不到要审核的图片或文本。

## 常用模块类型

| `kind` | 适合的模块 | 输入口 | 输出口 |
| --- | --- | --- | --- |
| `image_ocr` | OCR、图片文字识别 | 图片 | 文本 |
| `image_safety` | 图片合规检测、图片分类 | 图片 | 无 |
| `text_safety` | 文本合规检测、敏感词检测 | 文本 | 无 |
| `folder_processor` | 需要直接处理一个文件夹的模块 | 文件夹 | 默认无；可声明图片、文本或文件夹输出 |

如果模块要处理图片，优先选 `image_ocr` 或 `image_safety`。如果模块要处理文本，优先选 `text_safety`。如果模块要拿到一个文件夹路径，选 `folder_processor`。

## 自定义输出图片集合

模块可以自己声明额外的数据输出口。例如一个模块把输入图片处理成预览图、标注图或缩略图，再把这些新图片继续交给下游图片审核模块，就应该声明图片集合输出。

在 `module.json` 中声明：

```json
{
  "dataOutputs": [
    {
      "handle": "images",
      "name": "处理后图片",
      "dataType": "imageCollection"
    }
  ]
}
```

模块运行结束后，在 `output.json` 的 `outputs.images` 写出图片集合。下游模块可以直接从这个图片输出口连接到 `image_safety` 或 `image_ocr` 的图片输入口。

推荐连接：

```text
待测项目中所有图片 --图片数据--> 图片预处理模块
图片预处理模块 --处理后图片--> 图片合规检测
开始 -> 图片预处理模块 -> 图片合规检测 -> 输出结果
```

## image_ocr

`image_ocr` 有：

- 顺序输入
- 顺序输出
- 图片输入
- 文本输出

典型连接：

```text
开始 -> 图片文字识别 -> 文本审核 -> 输出结果
待测项目中所有图片 --图片数据--> 图片文字识别
图片文字识别 --文本数据--> 文本审核
```

OCR 模块要让后续文本模块拿到文字，需要在 `output.json` 里写 `results[].fullText` 或 `outputs.fullText`。具体写法见 [模块输入与输出说明](./模块输入与输出说明.md)。

## image_safety

`image_safety` 有：

- 顺序输入
- 顺序输出
- 图片输入

典型连接：

```text
开始 -> 图片合规检测 -> 输出结果
待测项目中所有图片 --图片数据--> 图片合规检测
```

图片审核模块通常直接输出审核结论，不向后续模块传数据。

## text_safety

`text_safety` 有：

- 顺序输入
- 顺序输出
- 文本输入

典型连接：

```text
开始 -> 文本合规检测 -> 输出结果
待测项目中所有文本 --文本数据--> 文本合规检测
```

如果文本来自 OCR，连接方式是：

```text
图片文字识别 --文本数据--> 文本合规检测
```

## folder_processor

`folder_processor` 有：

- 顺序输入
- 顺序输出
- 文件夹输入
- 如果模块声明了 `dataOutputs`，还会显示对应数据输出口

典型连接：

```text
开始 -> 文件夹处理模块 -> 输出结果
待审核文件夹 --文件夹数据--> 文件夹处理模块
```

模块运行时从 `input.json` 的 `inputs.folder.path` 读取文件夹路径。

如果文件夹处理模块会生成图片，可以把模块的图片输出口连接到图片审核模块：

```text
开始 -> 文件夹处理模块 -> 图片审核模块 -> 输出结果
待审核文件夹 --文件夹数据--> 文件夹处理模块
文件夹处理模块 --图片数据--> 图片审核模块
```

## 系统数据节点

客户端内置了一些数据节点，用来给模块提供图片、文本或文件夹。

| 节点 | 输出 |
| --- | --- |
| 待测项目中所有图片 | 图片集合 |
| 待测项目中所有文本 | 文本集合 |
| 产物文件夹中所有图片 | 图片集合 |
| 产物文件夹中所有文本 | 文本集合 |
| 待测项目相对路径下所有图片 | 图片集合 |
| 待测项目相对路径下所有文本 | 文本集合 |
| 待审核文件夹 | 文件夹 |
| 待审核文件夹下相对路径文件夹 | 文件夹 |
| 产物文件夹 | 文件夹 |
| 待产物文件夹下相对路径文件夹 | 文件夹 |
| 将两个图片集合合并 | 图片集合 |
| 将两个文本集合合并 | 文本集合 |

相对路径数据节点需要填写 `relativePath` 参数。例如只审核 `Assets/Icons` 下的图片，或把该目录作为文件夹传给模块，就填写：

```text
Assets/Icons
```

## 推荐流程模板

图片 OCR 加文本审核：

```text
开始
  -> 图片文字识别
  -> 文本合规检测
  -> 输出结果

待测项目中所有图片
  --图片数据--> 图片文字识别

图片文字识别
  --文本数据--> 文本合规检测
```

图片审核加文本审核并行：

```text
开始
  -> 图片合规检测
  -> 输出结果

开始
  -> 文本合规检测
  -> 输出结果

待测项目中所有图片
  --图片数据--> 图片合规检测

待测项目中所有文本
  --文本数据--> 文本合规检测
```

OCR、图片审核、文本审核组合：

```text
开始
  -> 图片文字识别
  -> 文本合规检测
  -> 输出结果

开始
  -> 图片合规检测
  -> 输出结果

待测项目中所有图片
  --图片数据--> 图片文字识别

待测项目中所有图片
  --图片数据--> 图片合规检测

图片文字识别
  --文本数据--> 文本合规检测
```

## 常见连接错误

| 现象 | 原因 | 处理 |
| --- | --- | --- |
| 模块运行了但处理数量为 0 | 没有连接数据线，或数据节点没有收集到文件。 | 给模块连接需要的图片、文本或文件夹数据线。 |
| 图片数据接不到文本模块 | 图片集合不能接文本输入。 | 先经过 OCR，或改用文本数据节点。 |
| OCR 后文本模块没有收到文字 | OCR 输出里没有 `fullText`。 | 在 `results[].fullText` 或 `outputs.fullText` 写识别文本。 |
| 自定义模块没有图片、文本或文件夹端口 | `kind` 写成了客户端不认识的值。 | 改成 `image_ocr`、`image_safety`、`text_safety` 或 `folder_processor`。 |
| 输出结果节点过早运行 | 顺序线没有表达完整依赖关系。 | 让输出结果节点等待所有关键审核步骤完成。 |
