# 模块开发快速开始

本文档用一个最小文本审核模块说明模块开发的完整路径。完成后，这个模块可以被 UGCAudit 客户端导入，并在流程中接收文本集合、输出审核结论和报告片段。

## 1. 创建模块文件夹

建议每个模块一个独立目录：

```text
DemoTextRiskModule/
  module.json
  main.py
  README.md
```

目录名可以自定，但根目录必须有 `module.json`。

## 2. 编写 module.json

把下面内容保存为 `DemoTextRiskModule/module.json`：

```json
{
  "id": "custom.demo.text-risk",
  "name": "示例文本审核",
  "kind": "text_safety",
  "summary": "读取文本集合，并标记包含风险词的内容。",
  "modelLabel": "无需模型",
  "icon": "file-check",
  "modelConfigured": true,
  "launch": {
    "launchType": "python",
    "command": "main.py",
    "args": ["--input", "{inputJson}", "--output", "{outputJson}"],
    "notes": "使用客户端自带 Python 运行 main.py。"
  },
  "parameters": [
    {
      "key": "riskWords",
      "name": "风险词",
      "description": "按行填写需要命中的词。",
      "parameterType": "stringList",
      "defaultValue": ["联系方式", "广告"],
      "required": false,
      "options": []
    }
  ]
}
```

这个模块的类型是 `text_safety`，所以它会在流程图上显示文本输入口。使用时要把文本数据节点或 OCR 模块的文本输出连到它。

## 3. 编写 main.py

把下面内容保存为 `DemoTextRiskModule/main.py`：

```python
import argparse
import json
from pathlib import Path


def load_json(path):
    return json.loads(Path(path).read_text(encoding="utf-8"))


def write_json(path, value):
    Path(path).write_text(json.dumps(value, ensure_ascii=False, indent=2), encoding="utf-8")


def collect_text_items(input_data):
    texts_port = input_data.get("inputs", {}).get("texts", {})
    return texts_port.get("items", [])


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--input", required=True)
    parser.add_argument("--output", required=True)
    args = parser.parse_args()

    input_data = load_json(args.input)
    params = input_data.get("params", {})
    risk_words = [str(item).strip() for item in params.get("riskWords", []) if str(item).strip()]
    text_items = collect_text_items(input_data)

    matches = []
    for item in text_items:
        text = str(item.get("text", ""))
        hit_words = [word for word in risk_words if word in text]
        if hit_words:
            matches.append({
                "name": item.get("name", ""),
                "path": item.get("path", ""),
                "hitWords": hit_words
            })

    verdict = "review" if matches else "pass"
    message = f"命中 {len(matches)} 个文本文件。" if matches else "未命中风险词。"
    report_lines = [
        "### 示例文本审核",
        "",
        f"- 处理文本：{len(text_items)}",
        f"- 命中文本：{len(matches)}",
        f"- 结论：{'需要人工复审' if matches else '通过'}",
        ""
    ]
    for match in matches[:20]:
        report_lines.append(f"- `{match['name']}` 命中：{', '.join(match['hitWords'])}")

    output = {
        "status": "completed",
        "verdict": verdict,
        "message": message,
        "processedFiles": len(text_items),
        "matchedFiles": len(matches),
        "artifactCount": 0,
        "matches": matches,
        "reportSection": "\n".join(report_lines)
    }
    write_json(args.output, output)


if __name__ == "__main__":
    main()
```

## 4. 在客户端导入

1. 打开客户端的“模块管理”。
2. 点击“导入模块”。
3. 选择 `DemoTextRiskModule` 文件夹。
4. 导入成功后，模块会出现在模块列表里。
5. 回到流程画布，把“示例文本审核”拖到流程中。
6. 给它连接顺序线，并给它连接文本数据线。
7. 运行审核任务。

## 5. 最小流程连接

这个模块需要文本输入。最简单的连接方式是：

```text
开始 -> 示例文本审核 -> 输出结果
待测项目中所有文本 --文本数据--> 示例文本审核
```

其中“待测项目中所有文本”到“示例文本审核”的连接是数据线；“开始”“示例文本审核”“输出结果”之间的连接是顺序线。

## 6. 运行后检查

每次运行时，客户端会在步骤目录里生成这些文件：

```text
input.json
output.json
stdout.log
stderr.log
result.md
artifacts/
```

检查重点：

- `input.json`：确认模块收到了期望的文件、参数和上游结果。
- `output.json`：确认模块写出了合法 JSON。
- `stderr.log`：确认没有启动错误、依赖缺失或异常堆栈。
- `result.md`：确认报告片段内容符合预期。
